# dashboard-panel skill

Scaffolds a new `src/plugins/<id>.ts` plugin for the Hive Command Center dashboard
from an endpoint name. The new panel self-registers on import and is automatically
placed by the adaptive layout engine via its `relevance(state)` function.

## Trigger

- "add a dashboard panel for X"
- "new hive panel hitting /v1/..."
- "/dashboard-panel <endpoint-path>"
- "scaffold a plugin for <endpoint>"

## What it generates

1. `src/plugins/<id>.ts` — a complete `PanelPlugin` implementation
2. One import line appended to `src/plugins/index.ts` (the self-register barrel)
3. `tests/plugins/<id>.test.ts` — relevance pure-fn unit tests
4. Runs `npm run build` hint + deploy reminder

## Template shape

Every generated plugin implements:
- `id`, `title`, `dataSources[]` — declarative: kind=poll/ws/state + endpoint + intervalKey
- `relevance(state: SystemState)` — pure fn returning { priority, size, weight? }
  - Default: priority 60, size 'md' when state has new data; 25 when idle
- `mount(el)` — one-time DOM build with panel-header + content container
- `update(state, budget)` — re-renders honoring budget (chartFps, graphMaxNodes, animate)
- `suspend()` / `resume()` — optional: pause timers/sims when hidden

## Usage

When the operator says "add a dashboard panel for /v1/lessons" or similar:

1. Derive a kebab-case `id` from the endpoint (e.g. `lessons`)
2. Derive a human `title` (e.g. `LESSONS`)
3. Determine `intervalKey` from the endpoint prefix:
   - `/board/*`   → `board` (10s)
   - `/v1/scout*` → `scout` (3s)
   - `/v1/*`      → `right` (30s)
   - `/v1/suno*`  → `suno`  (60s)
4. Write `src/plugins/<id>.ts` from the template below
5. Append `export * from './<id>.js';` to `src/plugins/index.ts`
6. Write a relevance unit test
7. Run `npm run build` (from ~/projects\hive-dashboard)
8. Run deploy: `xcopy /E /I /Y dist\ "%LOCALAPPDATA%\Lively Wallpaper\Library\wallpapers\hivecmd.v01\"`
9. Run `python ./hive/scripts/sync_skills.py`

## Panel template (fill in {ID}, {TITLE}, {ENDPOINT}, {INTERVAL_KEY})

```typescript
/**
 * plugins/{ID}.ts — {TITLE} panel.
 * Auto-generated by dashboard-panel skill.
 */
import { register } from './registry.js';
import type { PanelPlugin, RelevanceResult } from './contract.js';
import type { SystemState, RenderBudget } from '../state/types.js';
import { escHtml } from '../format.js';

function relevance(state: SystemState): RelevanceResult {
  // Adjust priority/size based on what matters to the operator:
  if (state.activity === 'idle') return { priority: 60, size: 'md' };
  return { priority: 25, size: 'sm' };
}

let _rootEl: HTMLElement | null = null;

function mount(el: HTMLElement): void {
  _rootEl = el;
  el.innerHTML = `
    <div class="panel-header">
      <span class="panel-label">{TITLE}</span>
    </div>
    <div id="v2-{ID}-content" class="{ID}-content">
      <p class="offline-state">Loading…</p>
    </div>
  `;
}

function update(_state: SystemState, budget: RenderBudget): void {
  if (!_rootEl) return;
  // TODO: render data from the last poll result
  // Use budget.chartFps, budget.animate, budget.chartMaxPoints to throttle
}

/** Called by the poll adapter when new data arrives from {ENDPOINT}. */
export function on{ID_PASCAL}Data(data: unknown): void {
  const content = document.getElementById('v2-{ID}-content');
  if (!content) return;
  // TODO: render data
  content.innerHTML = `<pre>${escHtml(JSON.stringify(data, null, 2).slice(0, 500))}</pre>`;
}

const plugin: PanelPlugin = {
  id:          '{ID}',
  title:       '{TITLE}',
  dataSources: [{ kind: 'poll', endpoint: '{ENDPOINT}', intervalKey: '{INTERVAL_KEY}' }],
  relevance,
  mount,
  update,
};

register(plugin);
export { plugin as {ID_CAMEL}Plugin };
```

## Relevance unit test template

```typescript
import { describe, it, expect } from 'vitest';
import { {ID_CAMEL}Plugin } from '../../src/plugins/{ID}.js';

function makeState(overrides = {}) {
  return { activity: 'idle', tier: 'idle', gatewayUp: true,
    tasks: { building: [], review: 0, qa: 0, ready: 0, done: 0 },
    escalations: { open: 0 }, resources: { gpus: [], cpuPct: 0, ramPct: 0, gaming: false, contended: false },
    counts: { costUsd: 0, tokRateHive: 0, tokRateClaude: 0, parseFailRate: 0 },
    ts: Date.now(), ...overrides } as any;
}

describe('{ID} plugin relevance', () => {
  it('returns a valid size hint', () => {
    const rel = {ID_CAMEL}Plugin.relevance(makeState());
    expect(['hero','lg','md','sm','min','hidden']).toContain(rel.size);
    expect(rel.priority).toBeGreaterThanOrEqual(0);
    expect(rel.priority).toBeLessThanOrEqual(100);
  });
});
```

## Notes

- The clock panel (`src/plugins/clock.ts`) is the canonical reference implementation.
- The escalations panel shows how `hidden` ↔ `hero` transitions work with the dwell bypass.
- Always run `npm test` after generating to verify no regressions.
- The skill self-syncs to the vault via `sync_skills.py` after use.
- Contract version: 2 (see `src/plugins/contract.ts`).
